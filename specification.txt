YouTube 多视角 Archive 同步观看聚合页
需求书 v0.1

1. 项目概述

1.1 项目背景

在观看 VTuber 联动的直播存档（Archive）时，用户常常希望同时查看多个成员的视角，以获得更完整的观看体验。
但现有 YouTube 原生页面不支持多视角聚合观看，用户若自行打开多个页面，会面临以下问题：

多个视频进度难以手动同步
多路声音重叠，难以集中管理
暂停、播放、跳转等操作需分别对多个页面执行
多视角切换和控制成本高，观看体验差

因此，本项目拟开发一个 面向 YouTube Archive 的多视角同步观看聚合页，让用户可在同一界面内同时加载多个 YouTube 视频，并统一进行播放控制、时间同步和音轨管理。


2. 项目目标

2.1 核心目标

提供一个可用的聚合观看页面，使用户能够：

同屏观看多个 YouTube Archive 视频
手动设置各视频相对于主时间轴的 offset
对多个视频进行统一播放/暂停/跳转控制
对每一路音轨单独进行 mute / volume / solo 控制
通过轻量的同步纠偏机制，减少时间漂移

2.2 产品定位

本产品定位为：

YouTube Archive 多视角观看工具
重度观看体验增强工具
面向手动配置、非自动识别的专业/半专业工具

2.3 非目标

本项目当前阶段不追求：

自动识别哪些视频属于同一场联动
自动分析并推导最佳 offset
支持正在直播中的 live 同步
支持广告处理
支持除 YouTube 外的其他视频平台
支持高级音频处理（如混音、压限、EQ、延迟补偿等）

3. 适用范围

3.1 当前支持范围

本项目仅支持：

YouTube 视频
可回放的 Archive / 普通视频
用户手动输入视频链接
用户手动设置 offset
聚合页内统一控制多个播放器

3.2 当前明确不支持范围

以下内容不在当前版本范围内：

YouTube Live
Twitch、Bilibili、Niconico 等平台
自动抓取用户当前浏览器 tab
自动配对同一场联动视频
自动识别广告并处理
音频波形分析同步
OCR、语音识别、弹幕分析等智能功能

4. 用户角色与使用场景

4.1 目标用户

目标用户包括但不限于：

有多视角观看需求的 VTuber 观众
想回看多人联动不同视角的用户
对观看控制体验有较高要求的重度用户

4.2 典型使用场景
场景 1：回看多人联动

用户有 3 个联动成员的 Archive 链接，希望：

同时观看三路画面
以其中一路为主视角
其他路静音或小声，仅保留主路声音
特定情况下手动调整各路音量观看其他视角
根据联动开场时间差，手动设置 offset
播放中自动保持大体同步
场景 2：对比不同视角反应

用户想观察多个成员在同一时刻的不同画面表现，希望：

快速跳转到某个时间点
所有路同时对齐到对应时刻
临时 solo 某一路音轨听反应
再切回主路音频
场景 3：手动精调同步

用户发现某一路略有漂移，希望：

修改其 offset
或通过系统自动小幅纠偏恢复同步
避免频繁手动 seek

7. 详细功能需求

7.1 视频源管理

7.1.1 添加视频源

系统应允许用户手动输入一个或多个 YouTube 视频链接。

输入形式
文本输入框输入 URL
一次输入一条
或支持批量输入（每行一个 URL，后续可选）

系统行为
校验是否为合法 YouTube 视频链接
从链接中解析 videoId
为每个 videoId 创建一个播放器实例
将该路视频加入当前会话
错误处理

若链接不合法：

显示错误提示
不创建播放器

若视频无法加载：

在该路播放器区域显示错误状态
不影响其他路正常使用

7.1.2 删除视频源

系统应允许用户删除某一路视频。

删除后行为
销毁对应播放器实例
清除该路本地状态
更新布局
若被删除的是主参考路，需要重新指定主参考路，或提示用户选择

7.1.3 视频源信息展示

每一路视频应展示基础信息：

路名称（默认可为 Video 1 / Video 2 / …）
视频标题（若可获取）
视频状态（未加载 / 已就绪 / 播放中 / 暂停 / 错误）
当前时间
offset

7.2 聚合页展示

7.2.1 同屏布局

系统应将多个播放器显示在同一页面内。

MVP 要求
默认使用简单 grid 布局
根据路数自动排列
支持至少 2~4 路的正常显示
后续可扩展
1 主 + N 辅布局
用户手动切换布局模式

7.2.2 单路展示区域

每一路视频区域应包含：

视频播放器画面
路名称
当前状态
当前播放时间
offset 显示与编辑控件
音频控制区
单路同步操作按钮（可选）

7.3 主参考时间轴与同步模型

7.3.1 主时钟概念

系统应基于统一的“主时间轴”进行同步，而不是依赖任意某一路播放器当前时间作为唯一真值。

说明

主时间轴用于：

计算所有视频应对齐到的目标时间
驱动 seek 和纠偏逻辑
提供全局同步参考

原则
系统内部应维护一个虚拟 master clock
每一路目标时间 = 主时间轴时间 + 该路 offset

7.3.2 主参考路

MVP 中系统应允许指定一条“主参考路”。

主参考路用途
用于初始化主时间轴
在用户进行全局 seek 时提供默认基准
作为用户理解会话同步关系的参考对象

后续说明

虽然底层同步应基于虚拟时钟，但 UI 层保留“主参考路”概念有助于用户理解。


7.4 全局播放控制

7.4.1 全局播放

系统应提供“全局播放”功能。

行为

用户点击全局播放后：

所有已就绪的视频开始播放
同步引擎进入运行状态
若某一路当前时间与目标时间差过大，可先校正再播放
异常

某一路播放失败时：

应显示该路错误
不应阻止其他路继续播放

7.4.2 全局暂停

系统应提供“全局暂停”功能。

行为

用户点击后：

所有正在播放的视频暂停
主时间轴停止推进
同步引擎暂停纠偏

7.4.3 全局 Seek / Sync Now

系统应提供“同步到当前目标时间”或“立即校正”功能。

行为

触发后：

计算每一路的目标时间
对偏差超出阈值的路执行 seek
对偏差较小的路可保留不动或做轻量纠偏
用途
用户修改 offset 后强制重新同步
某一路明显漂移后快速恢复
用户主动要求所有视频重新对齐

7.4.4 全局跳转

系统应允许用户输入或拖动一个全局目标时间，并让所有播放器跳转到该时刻附近。

行为

若全局目标时间为 T：

每一路应跳转至 T + offset

7.5 Offset 管理

7.5.1 手动设置 offset

每一路都应允许用户手动设置其相对于主时间轴的 offset。

表达方式

可用秒数表示：

正数：该路应比主时间轴更靠后
负数：该路应比主时间轴更靠前

例如：

+2.5s
-1.2s
MVP 要求
支持直接输入数字
支持小数秒

7.5.2 Offset 修改后的行为

用户修改某一路 offset 后：

系统应立即更新该路目标时间计算方式
若用户触发“Sync Now”，则按新 offset 对齐
是否自动立即 seek 可作为配置项，MVP 可先不自动跳转

7.5.3 Offset 显示

每一路应清晰显示当前 offset 值，避免用户混淆。


7.6 每路音频控制

这是本项目的重要核心功能之一。


7.6.1 Mute

每一路都必须提供 Mute 控制。

行为
开启 mute 后，该路静音
关闭 mute 后，该路恢复到其 volume 值对应的音量
mute 不应改变该路存储的 volume 数值，只改变输出状态

UI 要求
每一路有单独 mute 按钮
状态应清晰可见（例如高亮/图标变化）

7.6.2 Volume

每一路都必须提供 Volume Slider。

行为
用户可将该路音量调节到 0~100%
调整后应立即生效
volume 为该路持久状态之一
说明
volume = 0 不等同于 mute
mute 是一种状态开关
volume 是数值设定
MVP 范围
线性 slider 即可
不要求复杂音频曲线

7.6.3 Solo

每一路都必须提供 Solo 控制。

行为定义

当某一路开启 solo 时：

仅该路输出声音
其他所有路强制静音
其他路原本的 mute 状态与 volume 数值需被保留，不应被覆盖

当取消 solo 时：

系统恢复各路在进入 solo 前的正常音频状态
即恢复到各路原本的 mute + volume 生效结果
多路 Solo 规则

MVP 建议采用 单选 solo：

同一时刻只允许一路 solo
开启另一条路的 solo 时，自动取消原先的 solo

后续可扩展为多选 solo，但 MVP 不要求。

UI 要求
每一路有 solo 按钮
当前 solo 路应明显标识
进入 solo 模式时，系统应在全局区域显示“Solo Mode”

7.6.4 推荐附加功能：一键主视角有声

MVP 推荐提供一个快捷功能：

行为
主参考路音频恢复正常
其他所有路自动 mute

用途：

常见默认观看模式
联动多视角中最实用的模式之一

7.7 每路播放状态显示

每一路应至少展示以下状态：

未初始化
加载中
已就绪
播放中
暂停中
缓冲中
错误

这样用户可以快速判断：

哪一路未正常启动
哪一路发生异常
哪一路可能导致同步问题

7.8 时间漂移检测与纠偏

7.8.1 目标

在不依赖用户频繁手动调节的前提下，尽量保持各视频时间接近其目标时间。


7.8.2 同步检查周期

系统应以固定周期检查每一路视频当前时间与目标时间的差值。

建议
每 300ms ~ 1000ms 检查一次
MVP 可先从 500ms 开始

7.8.3 漂移判断

对每一路：

当前时间 = player 当前播放时间
目标时间 = masterTime + offset
差值 = 目标时间 - 当前时间

7.8.4 纠偏策略

MVP 应采用两级纠偏策略：

1）硬校正（seek）

当绝对误差大于某个较大阈值时：

直接执行 seek 到目标时间附近

建议阈值：

1.0s 左右
2）软校正（playbackRate）

当误差较小但仍需要修正时：

临时调整 playbackRate
误差修正后恢复到 1.0

建议：

playbackRate 控制在较小范围内，如 0.98 ~ 1.02

7.8.5 纠偏边界

系统不应频繁大幅改变 playbackRate，以免影响观看体验。

系统不应在误差极小时持续来回震荡修正，应设置一个稳定区间。

例如：

误差 < 100ms 时不处理
误差中等时软校正
误差过大时硬校正

7.8.6 用户可见状态

MVP 可选地显示每一路的同步状态：

已同步
轻微漂移
校正中
严重漂移

后续版本可进一步细化。


7.9 Session 管理（Milestone 2）

7.9.1 保存会话

允许用户保存当前配置，包括：

视频 URL 列表
各路名称
各路 offset
各路默认 volume
主参考路
布局配置（若有）

7.9.2 加载会话

允许用户恢复之前保存的多视角观看配置。


7.9.3 本地持久化

MVP 不强制要求。
Milestone 2 建议使用本地存储保存。


8. 明确不包含的功能

以下内容在当前需求范围内明确排除：


8.1 Live 支持

不支持正在直播的多视角同步。

原因：

live latency 不一致
各路 buffering 行为不同
同步逻辑复杂度显著上升

8.2 自动识别联动关系

不自动根据标题、频道、开播时间等识别哪些视频属于同一次联动。

原因：

规则复杂
命名不统一
非核心 MVP 价值

8.3 自动计算 offset

不通过音频分析、画面分析、字幕分析等方式自动推导 offset。

原因：

成本高
边界复杂
与 MVP 不匹配

8.4 广告处理

不在当前版本中处理广告带来的时间中断或状态异常。

前提假设：

用户已通过自身手段规避广告影响
或当前测试环境不将广告作为考虑对象

8.5 高级音频处理

不包含：

多路混音
音频总线
压缩器/限制器
EQ
声像控制
波形级对齐

8.6 浏览器插件控制现有 tab

当前版本不做“控制用户已打开 YouTube tab”的模式，仅做聚合页模式。


9. 异常与边界处理

9.1 单路视频加载失败
不影响其他路继续工作
该路显示错误状态
用户可移除或重试

9.2 某一路播放失败
记录该路错误状态
不阻塞其他路播放
用户可手动再次同步

9.3 某一路 seek 异常
标记该路未同步
允许用户点击 Sync Now 重试

9.4 用户设置极端 offset

例如：

数百秒 offset
超出视频时长范围

系统应：

允许输入，但在实际 seek 时做边界裁切
或提示目标时间无效

9.5 主参考路被删除

系统应：

自动选择一条新的主参考路，或
暂停同步并要求用户重新指定

11. 非功能需求


11.1 性能
在 2~4 路视频同时播放时，应保证基本可用
不要求无限扩展到大量视频路数
同步检查逻辑不能明显卡顿 UI


11.2 稳定性
单一路错误不应导致整个会话崩溃
同步引擎异常应可恢复


11.3 可维护性
播放器控制逻辑、同步引擎、UI 状态管理应分层
每一路播放器应为独立组件
同步策略应可替换/调整


11.4 可扩展性

系统设计应便于未来扩展：

session 保存
布局切换
热键
分享功能


12. 验收标准（MVP）

当满足以下条件时，可认为 MVP 基本完成：

12.1 基础能力


用户可输入多个 YouTube Archive 链接并成功加载
多个视频可在同一页面展示


12.2 同步能力


用户可为每一路设置 offset
点击播放后，多路视频能大致同步播放
点击暂停后，多路视频能同时暂停
点击 Sync Now 后，多路视频会重新对齐


12.3 音频能力

每一路都具备：

mute
volume
solo

且行为正确：

mute 仅控制静音状态，不覆盖 volume
volume 独立调节每一路音量
solo 启用后仅该路有声音，取消后恢复原状态


12.4 漂移控制
在正常观看过程中，多路视频不会迅速明显漂移
对小误差可进行轻量 playbackRate 纠偏
对大误差可通过 seek 拉回


12.5 容错
某一路异常时，其他路不崩溃
用户可以移除有问题的视频路

14. 一句话版本的产品定义

这是一个 面向 YouTube Archive 的多视角同步观看聚合页，允许用户手动配置多个视频的相对时间偏移，并在统一界面内实现同步播放控制、每路音轨管理和轻量时间纠偏，以提升联动存档的同屏观看体验。
